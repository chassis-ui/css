Chassis CSS includes many [CSS custom properties (variables)](https://developer.mozilla.org/en-US/docs/Web/CSS/Using_CSS_custom_properties) in its compiled CSS for real-time customization without the need to recompile Sass. These provide easy access to commonly used values like the theme colors, breakpoints, and primary font stacks when working in a browser inspector, code sandbox, or general prototyping.

**All custom properties are prefixed with `cx-`** to avoid conflicts with third-party CSS.

## Root variables

The variables below (note that `:root` is required) are available anywhere Chassis CSS is loaded. They're defined in `_root.scss` and included in the compiled dist files.

### Default

These CSS variables are available everywhere, regardless of color mode.

<Code lang="css" filePath="dist/css/chassis.css" fileMatch="(\[data-cx-theme=light\] {[^}]*})" />

### Dark mode

These variables are scoped to the built-in dark mode.

<Code lang="css" filePath="dist/css/chassis.css" fileMatch="(\[data-cx-theme=dark\] {[^}]*})" />

## Component variables

Chassis CSS uses custom properties as local variables for various components. This reduces compiled CSS size, prevents styles from leaking into nested elements (like nested tables), and makes it possible to restyle and extend components after Sass compilation without recompiling.

The [table documentation]([[docsref:/content/tables#how-do-the-variants-and-accented-tables-work]]) shows how Chassis uses CSS variables in practice. [Navbars]([[docsref:/components/navbar#css]]) and the [CSS grid system]([[docsref:/layout/css-grid]]) — primarily for gutters — also use CSS variables, with more component coverage planned.

Whenever possible, CSS variables are assigned at the base component level (e.g., `.navbar` for navbar and its sub-components). This reduces guessing on where and how to customize, and allows for straightforward modifications in future updates.

## Prefix

Most CSS variables use a prefix to avoid collisions with your own codebase. This prefix is in addition to the `--` that's required on every CSS variable.

Customize the prefix via the `$prefix` Sass variable. By default, it's set to `cx-` (note the trailing dash).

## Examples

CSS variables offer similar flexibility to Sass's variables, but without the need for compilation before being served to the browser. The snippet below resets the page's font and link styles with CSS variables.

```css
body {
  font: 1rem/1.5 var(--cx-font-sans-serif);
}
a {
  color: var(--cx-blue);
}
```

## Focus variables

Chassis CSS provides custom `:focus` styles using a combination of Sass and CSS variables that can be optionally added to specific components and elements. Chassis does not yet globally override all `:focus` styles.

Default values are set in Sass and can be customized before compiling.

<ScssDocs name="focus-ring-variables" file="scss/config/_defaults.scss" />

Those variables are then reassigned to `:root` level CSS variables that can be customized in real-time, including with options for `x` and `y` offsets (which default to their fallback value of `0`).

## Grid breakpoints

Grid breakpoints are available as CSS variables (except `xs`), but **CSS variables do not work in media queries**. This is by design in the CSS spec, but may change with future support for `env()` variables. See [this Stack Overflow answer](https://stackoverflow.com/a/47212942) for context. In the meantime, these variables work in other CSS situations and in JavaScript.
