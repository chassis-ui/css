## Examples

Shadow utilities add or remove `box-shadow` using a scale of elevation levels. The `$enable-component-shadows` Sass variable controls whether component shadows are enabled globally; utility classes apply shadows independently of that setting.

<Example class="overflow-hidden" code={`<div class="shadow-none rounded p-medium mb-xlarge">No shadow</div>
<div class="shadow-small rounded p-medium mb-xlarge">Small shadow</div>
<div class="shadow rounded p-medium mb-xlarge">Medium shadow</div>
<div class="shadow-large rounded p-medium mb-xlarge">Large shadow</div>
<div class="shadow-inset rounded p-medium mb-medium">Inset shadow</div>
<div class="shadow-emboss rounded p-medium">Emboss shadow</div>`} />


### Elevation scale

Elevation utilities map to design token names.

    })
  }
</div>

## Color shadows

Shadow utilities support color tinting through a CSS variable mechanism — a transparent `--cx-shadow-color` channel is built into each elevation class, and a color utility activates it.

### How it works

Each elevation class defines `--cx-shadow-color: transparent` alongside the shadow value. Color shadow classes (`.shadow-primary`, etc.) set `--cx-shadow-color` using CSS relative color syntax derived from the context color variable. The transparent shadow layer then becomes visible with a tint. This approach avoids generating every color × elevation combination at compile time.

```css
.shadow-50 {
  --cx-shadow-color: transparent;
  box-shadow: 0rem 0.5rem 2rem -0.5rem rgba(0, 0, 0, 0.1), 0rem 0.15rem 1rem -0.15rem var(--cx-shadow-color, transparent), 0rem 0.15rem 1rem -0.15rem rgba(0, 0, 0, 0.1) !important;
}
.shadow-primary {
  --cx-shadow-color: oklch(from var(--cx-primary) l c h / 0.1);
}
```

<Callout type="warning">
  While this way is not yet in direct control of designer but helps to avoid increasing file size by creating every color shadows including their hover states.
</Callout>

### Color tinting

Color tinting is applied by combining an elevation class with a context color shadow class.

<div class="cxd-example">
  {
    getData('color-values').map((value, index) => {
      return <p class={`shadow-primary shadow-${value.name} p-medium rounded`} style={`margin-bottom: calc(1rem + (.25rem * ${index}))`}>This is <code>$cx-shadow-elevation-primary-{value.name}</code> shadow.</p>
    })
  }
</div>

<Code lang="html" code={getData('color-values').map((value) => `<p class="shadow-primary shadow-${value.name}">...</p>`).join('\n')} />

## Hover shadows

Appending `-hover` to any shadow class applies the shadow only on `:hover`.

<Example code={`<p class="shadow-30-hover p-medium rounded">This is a hover shadow.</p>
<p class="shadow-primary-hover shadow-50 p-medium rounded">This is a shadow with primary hover.</p>`} />
