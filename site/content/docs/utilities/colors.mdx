Color utilities cover two sub-systems: context colors and opacity control. Both integrate with the [color system]([[docsref:/core-concepts/color-system]]).

## Color

Color utilities set the CSS `color` property using the Chassis context color palette. They do not set `background-color`, so `.bg-*` [background utilities]([[docsref:/utilities/background]]) may be needed for colored regions. For link colorization with interactive states (`:hover`, `:focus`, `:active`, `:visited`), see the [`.link-*` utilities]([[docsref:/utilities/link]]).

### Body colors

Body color classes draw from the `default` context palette.

<Example code={`<p class="fg-main">.fg-main</p>
<p class="fg-subtle">.fg-subtle</p>
<p class="fg-slight">.fg-slight</p>
<p class="fg-inverse bg-inverse">.fg-inverse</p>
<p class="fg-solid">.fg-solid</p>
<p class="fg-highlight">.fg-highlight</p>`} />

### Context foreground colors

Context colors are available as foreground utilities via the `.fg-{context}` class pattern.

<Example code={[...getData('context-colors').map((color) => `<p class="fg-${color.name} bg-contrast">.fg-${color.name}</p>`)]} />

### Context palette colors

Each context color exposes a set of foreground roles through a `{context}-fg-{role}` prefixed class pattern.

<Example code={`<p class="primary-fg-main">.primary-fg-main</p>
<p class="primary-fg-subtle">.primary-fg-subtle</p>
<p class="primary-fg-slight">.primary-fg-slight</p>
<p class="primary-fg-inverse primary-bg-inverse">.primary-fg-inverse</p>
<p class="primary-fg-highlight">.primary-fg-highlight</p>`} />

## Opacity

Opacity utilities control the alpha channel of any context color foreground through the `--cx-fg-opacity` custom property.

### How it works

Each `.fg-{context}` class writes an OKLCH relative color expression into `--cx-fg-color`, using `--cx-fg-opacity` as the alpha channel. A shared selector applies `color: var(--cx-fg-color)`. `--cx-fg-opacity` defaults to `1`.

```css
.fg-primary {
  --cx-fg-color: oklch(from var(--cx-primary) l c h / var(--cx-fg-opacity, 1));
}

/* shared selector */
.fg-primary, .fg-secondary, /* … */ {
  color: var(--cx-fg-color);
}

.fg-opacity-50 {
  --cx-fg-opacity: 0.5;
}
```

### Opacity override

Override `--cx-fg-opacity` with an inline style or a `.fg-opacity-*` utility class.

<Example code={`<div class="fg-primary">This is default primary text</div>
<div style="--cx-fg-opacity: .6;" class="fg-primary">This is 60% opacity primary text</div>`} />

Or, choose from any of the `.fg-opacity-*` utilities:

<Example code={getData('token-opacities').map((opacity) => `<p class="fg-primary fg-opacity-${opacity.name}">fg-opacity-${opacity.name}</p>`)} />

### Context opacities

`.fg-opacity-subtle` and `.fg-opacity-slight` match the alpha values of the pre-computed `fg-subtle` and `fg-slight` body color tokens, and can be applied to any context color.

<Example code={`<div class="fg-primary fg-opacity-subtle p-2xsmall">fg-opacity-subtle</div>
<div class="fg-primary fg-opacity-slight p-2xsmall">fg-opacity-slight</div>`} />

## Reset color

`.fg-reset` removes any foreground color override, causing the element to inherit its color from the nearest ancestor.

<Example code={`<p class="fg-subtle">
    Secondary body text with a <a href="#" class="fg-reset">reset link</a>.
  </p>
  <p class="fg-subtle">
    Secondary body text with a <a href="#">normal link</a>.
  </p>`} />

## CSS

Foreground color utilities are generated from Sass maps built on design token values via the utilities API.

### Reference

All foreground color utilities set `color` through the shared `--cx-fg-color` custom property. Context color foregrounds embed `--cx-fg-opacity` in the color expression; body and palette foregrounds reference a pre-computed token and are not affected by opacity utilities.

### Sass variables

Foreground color tokens are part of the color system [color system]([[docsref:/core-concepts/color-system]]) in `scss/tokens/` and not listed here.

### Sass maps
