Background utilities cover three sub-systems: context color backgrounds, a gradient overlay, and opacity control. All three integrate with the [color system]([[docsref:/core-concepts/color-system]]).

## Background color

Background utilities set `background-color` using the Chassis context color palette. They do not set `color`, so `.fg-*` [color utilities]([[docsref:/utilities/colors]]) may be needed for readable text.

### Body backgrounds

Body background classes draw from the `default` context palette.

<Example code={`<p class="bg-main p-2xsmall">.bg-main</p>
<p class="bg-even">.bg-even</p>
<p class="bg-evident">.bg-evident</p>
<p class="bg-highlight">.bg-highlight</p>
<p class="bg-solid fg-solid">.bg-solid</p>
<p class="bg-inverse fg-inverse">.bg-inverse</p>`} />

### Context backgrounds

Context color backgrounds follow the `.bg-{context}` class pattern, where `{context}` is any registered context color.

<Example code={getData('context-colors').map((contextColor) => `<p class="bg-${contextColor.name } fg-contrast p-2xsmall">.bg-${contextColor.name}</p>`)} />

### Palette backgrounds

Each context color exposes the full body background scale through a `{context}-bg-{role}` prefixed class pattern.

<Example code={`<p class="primary-bg-main">.primary-bg-main</p>
<p class="primary-bg-even">.primary-bg-even</p>
<p class="primary-bg-evident">.primary-bg-evident</p>
<p class="primary-bg-highlight">.primary-bg-highlight</p>
<p class="primary-bg-solid primary-fg-solid">.primary-bg-solid</p>
<p class="primary-bg-inverse primary-fg-inverse">.primary-bg-inverse</p>`} />

### Dimmed backgrounds

Dim classes apply semi-transparent background colors from the context palette. The opacity levels — `main`, `subtle`, and `slight` — are defined in the [dim opacity scale](#context-opacities).

<Example class="cxd-pattern-dots" code={`<p class="dim-main fg-inverse">.dim-main</p>
<p class="dim-subtle">.dim-subtle</p>
<p class="dim-slight">.dim-slight</p>`} />

<Example class="cxd-pattern-dots" code={`<p class="primary-dim-main primary-fg-inverse">.primary-dim-main</p>
<p class="primary-dim-subtle">.primary-dim-subtle</p>
<p class="primary-dim-slight">.primary-dim-slight</p>`} />

## Background gradient

`.bg-gradient` adds a linear gradient overlay — a semi-transparent white that fades to transparent at the bottom — to any background. The gradient is exposed as a CSS custom property: `--cx-bg-gradient`.

The gradient is also available in custom CSS via `background-image: var(--cx-bg-gradient)`.

<Example code={getData('context-colors').map((contextColor) => `<p class="bg-${contextColor.name } bg-gradient fg-contrast p-xsmall">.bg-${contextColor.name}.bg-gradient</p>`)} />

## Opacity

Opacity utilities control the alpha channel of any context color background through the `--cx-bg-opacity` custom property.

### How it works

Each `.bg-{context}` class writes an OKLCH relative color expression into `--cx-bg-color`, using `--cx-bg-opacity` as the alpha channel. A shared selector applies `background-color: var(--cx-bg-color)`. `--cx-bg-opacity` defaults to `1`.

```css
.bg-primary {
  --cx-bg-color: oklch(from var(--cx-primary) l c h / var(--cx-bg-opacity, 1));
}

/* shared selector */
.bg-primary, .bg-secondary, /* … */ {
  background-color: var(--cx-bg-color);
}

.bg-opacity-50 {
  --cx-bg-opacity: 0.5;
}
```

### Opacity override

Override `--cx-bg-opacity` with an inline style or a `.bg-opacity-*` utility.

<Example class="cxd-pattern-dots" code={`<div class="bg-primary p-2xsmall primary-contrast-color">
  This is default primary background</div>
<div style="--cx-bg-opacity: .4;" class="bg-primary p-2xsmall">
  This is 40% opacity primary background</div>`} />

Or, choose from any of the `.bg-opacity-*` utilities:

<Example class="cxd-pattern-dots" code={getData('token-opacities').map((opacity) => `<div class="bg-primary bg-opacity-${opacity.name } p-2xsmall">This is <code>bg-opacity-${opacity.name}</code> background</div>`)} />

### Context opacities

The three dim opacity levels (`main`, `subtle`, `slight`) are also available as `.bg-opacity-main`, `.bg-opacity-subtle`, and `.bg-opacity-slight`, and can be applied to any background color.

<Example class="cxd-pattern-dots" code={`<div class="bg-primary bg-opacity-main p-2xsmall">This is <code>dim-main</code> opacity primary background</div>
<div class="bg-primary bg-opacity-subtle p-2xsmall">This is <code>dim-subtle</code> opacity primary background</div>
<div class="bg-primary bg-opacity-slight p-2xsmall">This is <code>dim-slight</code> opacity primary background</div>`} />

## CSS

Background utilities are generated from Sass maps built on design token values via the utilities API.

### Reference

All background utilities set `background-color` through the shared `--cx-bg-color` custom property. Context color backgrounds embed `--cx-bg-opacity` in the color expression; body, palette, and dim backgrounds reference a pre-computed token and are not affected by opacity utilities.

### Sass variables

Background color tokens are part of the [color system]([[docsref:/core-concepts/color-system]]) in `scss/tokens/` and not listed here.

`$bg-gradient` defines the `background-image` value applied by `.bg-gradient` and by the `gradient()` mixin when `$enable-background-gradients` is `true`.

<ScssDocs name="variable-gradient" file="scss/config/_defaults.scss" />

### Sass maps

Background color maps are part of the [color system]([[docsref:/core-concepts/color-system]]) in `scss/maps/` and not listed here. Background opacity utilities are generated from two Sass maps in `scss/maps/_opacities.scss`.

`$opacities` defines the full token-named opacity scale for `.bg-opacity-{level}` utilities.

<ScssDocs name="opacity-levels-map" file="scss/maps/_opacities.scss" />

`$dim-opacities` defines three semantic dim levels.

<ScssDocs name="dim-opacities-map" file="scss/maps/_opacities.scss" />
