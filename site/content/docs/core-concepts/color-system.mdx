import { sassVars } from '@libs/sass'

Chassis CSS organizes color into three layers: **primitive colors** give each brand hue a numbered scale, **context colors** map those scales to semantic roles, and **body colors** expose them as the CSS variables components use directly. To switch a theme or context, only the active palette needs to change — all components update automatically.

## Primitive Colors

Primitive colors and their scale values are defined in Chassis Tokens. By default there are seven hues, one per brand role — and any project can swap them for its own palette.

<div class="overflow-scroll text-nowrap mb-medium">
  <div class="grid font-code" style="--cx-grid-columns: 11; --cx-grid-gap: var(--cx-space-2xsmall);">
  {
    getData('colors').map((color) => {
      return (
        <div class="grid" style="width: 12rem; --cx-grid-columns: 1; grid-template-rows: auto;">
          <div class={`p-medium rounded-small mb-xsmall position-relative swatch-${color.name} bg-${color.name} fg-contrast`}>
            <strong class="d-block">{color.name}</strong>
            {sassVars('color')?.[color.name]}
          </div>
          <div class={`p-medium rounded-small cxd-${color.name}-05`}>{color.name}-05</div>
            {
              Array.from({ length: 9 }, (_, i) => {
                const value = (i + 1) * 10; // Generates 10, 20, ..., 90
                return <div class={`p-medium rounded-small cxd-${color.name}-${value}`}>{color.name}-{value}</div>
              })
            }
          <div class={`p-medium rounded-small cxd-${color.name}-95`}>{color.name}-95</div>
        </div>
      )
    })
  }
  </div>
</div>

Each hue has eleven scale steps: `05`, `10`–`90` in increments of 10, and `95`, with `50` as the mid-point. Components and utilities don't reference these scales directly — they're exposed as CSS variables (e.g. `--cx-primary-50`) if you need them in custom work.

## Context Colors

There are **eleven context colors** in total: the seven brand hues, plus `default`, `alternate`, `black`, and `white`. These are the colors you can use with the `.context` class and all context-aware components.

<div class="row g-xsmall mb-medium font-code">
  {
    getData('context-colors').map((color) => {
      return (
        <div class="medium:col-4">
          <div class={`p-medium m-0 bg-${color.name} fg-contrast rounded-small`}>--cx-{color.name}</div>
        </div>
      )
    })
  }
</div>

The four non-brand context colors each have a specific role:

- **`default`** — the standard page background and text. In dark mode it inverts automatically: light background with dark text becomes dark background with light text.
- **`alternate`** — for prominent regions like sidebars, banners, or callout sections. Whether it inverts in dark mode is controlled by the Chassis Tokens configuration.
- **`black`** and **`white`** — fixed values that stay the same in both light and dark mode.

### Context Palettes

Each context color comes with a full palette of **36 semantic tokens** grouped into nine categories. Each token maps to a CSS variable: `--cx-{context}-{token}` — for example, `--cx-primary-fg-main`.

<DocsContextPalette />

Palette values are resolved at build time by Chassis Tokens, using Tokens Studio's color processor. Each shade combines a primitive color reference with an opacity value — both defined as tokens. The output arrives in CSS as pre-computed custom properties, ready to use. The example below shows the derivation pattern:

```scss
// Illustrative — derivation happens in Chassis Tokens at build time
// Light theme
$default-fg-main:      $black;
$default-fg-subtle:    to-opacity($default-fg-main, $opacity-fg-subtle);
$default-fg-slight:    to-opacity($default-fg-main, $opacity-fg-slight);
$default-bg-main:      $white;
$default-bg-even:      $neutral-05;
$default-bg-evident:   $neutral-10;
```

### Body Colors

The body context is set in Chassis Tokens — `default` by default. That context's palette becomes the page-level palette: the context prefix is dropped, so `--cx-default-fg-main` becomes `--cx-fg-main`, `--cx-default-bg-main` becomes `--cx-bg-main`, and so on.

From there, each palette token gets one more alias — a **body semantic variable** that maps directly to the CSS property it controls:

<DocsBodyPalette />

The [Context Class]([[docsref:/core-concepts/context-class]]) is how you trigger a palette switch in markup. Add `.context primary` to any element and its body variables instantly shift to the `primary` palette — for that element and everything inside it.

## Further Reading

Primitive colors, scale values, opacity levels, and context mappings are all configured in Chassis Tokens. The CSS variable names in Chassis CSS match the Figma variable paths in Chassis Figma — `color/context/{context}/{token}` in Figma becomes `--cx-{context}-{token}` in CSS, so designs and code always point to the same value.

For applying colors in markup, see the [Colors]([[docsref:/utilities/colors]]) and [Background]([[docsref:/utilities/background]]) utility pages. To build context-aware components, see the [Context Class]([[docsref:/core-concepts/context-class]]) page.
