The `.context` class is the mechanism Chassis CSS uses to apply color palettes to components. Rather than writing explicit color values, components reference CSS custom properties such as `--cx-fg-main` and `--cx-bg-main` that are re-assigned when a context changes. Any component placed inside a `.context` block automatically adapts its colors to match the new palette.

## How It Works

Chassis CSS does not write color properties directly on component classes. Instead, it uses <a href="https://www.w3.org/TR/css-variables-1/#example-c3e4e8d9" target="_blank">repeating CSS custom properties</a> and extends Sass placeholder classes to share property definitions while keeping values flexible.

### Placeholder Selector

The `%context` placeholder selector iterates over all context colors using the [context mixin](#context-mixin) to generate one palette class per color. These selectors do not set any CSS property values — they only reassign the context CSS custom properties.

### Component Class

The `.context` class extends the above placeholder selectors to generate a class for every context color and style combination.

<ScssDocs name="context-class" file="scss/_context.scss" />

This generates all context color and style variants ready to use.

<Example code={`<p class="context primary">This is primary basic context.</p>
<p class="context primary solid">This is primary solid context.</p>
<p class="context primary smooth">This is primary smooth context.</p>
<p class="context primary outline">This is primary outline context.</p>`} />

Although `default`, `alternate`, `neutral`, `black`, and `white` may look similar at first glance, they serve distinct semantic roles and behave differently across light and dark modes. `default` is the standard body context — light background with dark text. `alternate` is for emphasized regions such as sidebars or banners, and may not invert in dark mode. `neutral` is a muted, low-emphasis surface. `black` and `white` are absolute and do not adapt to light or dark mode at all.

<Example code={`<p class="context default border">This is default basic context.</p>
<p class="context alternate border">This is alternate basic context.</p>
<p class="context neutral border">This is neutral basic context.</p>
<p class="context black border">This is black basic context.</p>
<p class="context white border">This is white basic context.</p>`} />

## Usage

### Context Colors

Adding the color name to `.context` switches the active palette. Each context color provides coordinated foreground, background, border, icon, and link values.

<Example code={[`<p class="p-xsmall">
    <span class="icon icon-info-circle-solid"></span>
    This is body context <a href="#">with link</a>, same as default.
  </p>`].concat(getData('context-colors').map((color) => `<p class="context ${color.name} p-xsmall">
    <span class="icon icon-info-circle-solid"></span>
    This is ${color.name} context <a href="#">with link</a>.
  </p>`))} />

### Context Styles

Adding a style modifier controls which palette tokens are mapped to each visual property. Four styles are available:

- **`basic`** (default) — inherits the ambient background; switches foreground, border, and icon colors to the context.
- **`solid`** — fills with the context’s primary background color; uses the inverted foreground for maximum contrast.
- **`smooth`** — fills with a tinted, low-opacity variation of the context background; retains readable foreground contrast.
- **`outline`** — transparent background; context color applied as a border and foreground tint.

<Example code={`<p class="context primary p-xsmall">
    <span class="icon icon-info-circle-solid"></span>
    This is a basic context component <a href="#">with link</a>.
  </p>
  <p class="context primary solid p-xsmall">
    <span class="icon icon-info-circle-solid"></span>
    This is a solid context component <a href="#">with link</a>.
  </p>
  <p class="context primary smooth p-xsmall">
    <span class="icon icon-info-circle-solid"></span>
    This is a smooth context component <a href="#">with link</a>.
  </p>
  <p class="context primary outline p-xsmall">
    <span class="icon icon-info-circle-solid"></span>
    This is an outline context component <a href="#">with link</a>.
  </p>`} />

## Custom Components

### Custom Property with Fallback

An `.example` component uses a custom property with a built-in fallback chain. This lets you override the color where needed and fall back to context-aware values everywhere else.

<ScssDocs name="example-1" file="site/src/scss/_context-example.scss" />

The custom property can be overridden inline, falls back to `--cx-fg-highlight`, and ultimately adapts to the active context.

<Example code={`<div>
    <p class="example">--example-fg</p>
  </div>

  <div style="--example-fg: var(--cx-fg-highlight);">
    <p class="example">--example-fg is --cx-fg-highlight</p>
  </div>

  <div style="--example-fg: var(--no-color);">
    <p class="example">--example-fg falls back to --cx-fg-subtle</p>
  </div>

  <div class="context primary">
    <p class="example">--example-fg</p>
  </div>

  <div class="context primary" style="--example-fg: var(--no-color);">
    <p class="example">--example-fg falls back to --cx-fg-subtle</p>
  </div>

  <div class="context primary">
    <p>--cx-fg-main</p>
  </div>`} />

### Intercepting Style Overrides

`.example-2` is a context component that uses `--cx-fg-subtle` for its `color` property. A component-level custom property reserves a slot for future manipulation.

<ScssDocs name="example-2" file="site/src/scss/_context-example.scss" />

The `%solid-context` placeholder overrides `--cx-fg-color` with the new `--cx-fg-main`. We intercept this by setting `--cx-fg-subtle` as `--cx-fg-color` before the override takes effect.

<Example code={`<p class="example-2">--cx-fg-subtle</p>
<p class="example-2 primary">--cx-fg-subtle</p>
<p class="example-2 primary solid">--cx-fg-subtle</p>`} />

The `%solid-context` selector overrides the component colors.

### Full Context Component

`.example-3` is a fully context-aware component that maps component-scoped custom properties to body colors and supports the solid style modifier.
