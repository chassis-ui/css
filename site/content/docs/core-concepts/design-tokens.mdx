Every color, spacing step, border radius, font size, and shadow in Chassis CSS comes from a token. Nothing is hardcoded. This means a single change in Chassis Tokens flows out to every component and utility that references it вЂ” in CSS, in Figma, and across every platform you build for.

## How Tokens Are Structured

Tokens are layered. The lower layers hold raw values; the higher layers turn those values into named design decisions.

<CxTable>
| Layer | Token set files | What it contains |
| --- | --- | --- |
| **Metric** | `base/metric-base`, `base/metric-px` | Unit scale, spacing scale, opacity levels |
| **Brand** | `base/brand-base`, `brand-{name}/brand-base` | Primitive color palettes, typography (family, weight, size, line-height) |
| **Theme** | `base/theme-base`, `base/theme-light`, `base/theme-dark` | Semantic context color palettes for light and dark modes |
| **App** | `base/app-base`, `base/app-{name}` | Component-level metrics: spacing, color, font, shadow, grid, border |
| **Effect** | `base/effect-base` | Shadow definitions |
| **Motion** | `base/motion-base` | Duration and easing |
</CxTable>

A token build composes these layers вЂ” brand Г— theme Г— app вЂ” and resolves the result for a specific platform and screen size.

## Collections and Modes

In Figma, token layers map to variable collections. Each collection has modes, so one token can hold different values depending on what's active вЂ” a brand, a theme, or an app. The four collections are:

<CxTable>
| Collection | Modes |
| --- | --- |
| **Brand** | One per brand (e.g. `chassis`, `test`) |
| **Theme** | `light`, `dark` |
| **App** | One per app (e.g. `docs`, `demo`) |
</CxTable>

Collections are switched independently. Changing the Theme from `light` to `dark` updates every component that references a theme token вЂ” without touching any Brand or App settings.

## What's Inside

Here's an overview of each token category and how it shows up in CSS.

### Color

Color tokens follow the three-layer structure of the [Color System]([[docsref:/core-concepts/color-system]]):

- **`color/palette/{hue}/{step}`** вЂ” the primitive scale (e.g. `color/palette/primary/50`)
- **`color/context/{context}/{token}`** вЂ” the semantic palette for each context color (e.g. `color/context/primary/fg-main`)
- **Component color definitions** in `app` layer вЂ” per-component values that reference context tokens (e.g. `color/button/primary/bg-idle`)

A context token becomes `--cx-{context}-{token}` in CSS. The body shorthand `--cx-fg-main` is an alias for the active body context's `fg-main`.

### Spacing

Spacing uses a named scale rather than numeric steps. It runs from `zero` to `6xlarge`:

`zero` В· `4xsmall` В· `3xsmall` В· `2xsmall` В· `xsmall` В· `small` В· `medium` В· `large` В· `xlarge` В· `2xlarge` В· `3xlarge` В· `4xlarge` В· `5xlarge` В· `6xlarge`

The global scale lives under `space/context/{step}` in Figma and becomes `--cx-space-{step}` in CSS. Component-level spacing (like button padding) lives under `space/{component}/{variant|size}/{property}` and maps to `--cx-{component}-{property}`.

### Typography

Brand tokens define the font family, weight scale, and size scale. App tokens then compose these into named text styles: `font/{family}/{size}/{weight}`. The resulting CSS variables follow the `--cx-font-{property}` pattern.

### Metrics

Component metrics вЂ” border radius, border width, icon size, grid columns, and gap вЂ” live in `app-base`. Global values become `--cx-{property}`; component-specific ones become `--cx-{component}-{property}`.

### Motion

Duration and easing tokens live in `base/motion-base` and are exposed as `--cx-motion-{property}`. Use them to keep transitions consistent across components.

## Figma and CSS Variables

Chassis Tokens and Chassis CSS use the same naming structure, so a token in Figma and its CSS variable always point to the same design decision:

```
color/context/{context}/{token}  в†’  --cx-{context}-{token}
space/context/{step}             в†’  --cx-space-{step}
```

For example, `color/context/primary/fg-main` in Figma is `--cx-primary-fg-main` in CSS вЂ” the same value, used by the `fg-primary` utility class and the `.context.primary` palette.

## Toolchain

Tokens are authored with [Tokens Studio](https://tokens.studio) in Figma and synced to the `chassis-tokens` repository as JSON. [Style Dictionary](https://amzn.github.io/style-dictionary/) then transforms them into platform-specific output: SCSS for web, Swift for iOS, and XML for Android. See the Chassis Tokens repository for build instructions and configuration details.
