Chassis CSS components follow a base-and-modifier pattern. Properties shared by every variant live on a base class such as `.button`. Properties unique to a variant live on a modifier class such as `.button.primary` or `.button.success`. Modifier classes are generated by looping over Sass maps, so adding, renaming, or removing a variant is a single edit to a map.

## Base and modifier classes

Each component starts with a base class that holds the shared layout, typography, spacing, and state styles:

```scss
// Base class — shared by every variant
.button {
  // typography, padding, border, focus styles, etc.
}
```

Modifier classes layer on top, supplying only the values that change between variants. For most components those values come from a context color palette, so the modifiers are generated by iterating over the `$context-colors` Sass map:

```scss
// Modifier classes — generated from $context-colors
@each $color, $value in $context-colors {
  .button.#{$color} {
    @include button-variant($color);
  }
}
```

Customize the map and recompile, and the loop produces a different set of modifiers automatically. See [Sass maps and loops]([[docsref:/customize/sass#maps-and-loops]]) for the full pattern.

## Building your own components

The pattern is meant to be reused. The custom callouts in this documentation, for instance, are a base class plus three modifier classes:

<div class="cxd-example">
  <div class="cxd-callout my-0">
    <strong>This is a callout.</strong> Built specifically for these docs so important notes stand out. It has three variants applied through modifier classes.
  </div>
</div>

```html
<div class="callout">…</div>
```

The CSS keeps the bulk of the styling on `.callout` and limits each modifier to the property that changes — in this case a single `border-left-color`:

```scss
// Base class
.callout { … }

// Modifier classes
.callout.info { border-left-color: var(--cx-info-border-main); }
.callout.warning { border-left-color: var(--cx-warning-border-main); }
.callout.danger { border-left-color: var(--cx-danger-border-main); }
```

Combining the base class with one modifier yields a complete variant family:

<Callout>
**This is an info callout.** Example text to show it in action.
</Callout>

<Callout type="warning">
**This is a warning callout.** Example text to show it in action.
</Callout>

<Callout type="danger">
**This is a danger callout.** Example text to show it in action.
</Callout>

When you build your own components, follow the same convention: keep shared rules on the base class, drive variants from a Sass map, and let `@each` produce the modifier classes for you.
